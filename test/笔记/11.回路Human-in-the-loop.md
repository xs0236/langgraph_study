# 代码文件 content_factory_hitl.py

Human-in-the-loop（人在回路）是 LangGraph 非常实用的特性。核心思路就是：图执行到某个节点时主动"暂停"，把中间结果抛给人类审核，等人类输入决策后再继续执行。
下面我按「概念 → 最小示例 → 结合你的项目」三步来讲。
一、核心概念：interrupt
Python
from langgraph.types import interrupt, Command

def human_review_node(state): # 执行到这里，整个图会暂停！状态被 checkpointer 保存
decision = interrupt({
"draft": state["draft"], # 把当前状态传给前端展示
"message": "请审核这篇草稿" # 提示信息
}) # 当外部调用 Command(resume=...) 后，这里才会继续
return {"approved": decision == "y"}

动作 代码 说明

from langgraph.types import interrupt, Command

def human_review(state):
"""文章写完后暂停，等用户输入"""
decision = interrupt({
"stage": "article_review",
"article": state.get("article", ""),
"edit_feedback": state.get("edit_feedback", ""),
"message": "文章已生成，请输入操作：continue(继续) / rewrite(重写) / stop(终止)"
})
return {"human_decision": decision}

def m_s(s):
if not s.get("article"):
return {"next_agent": "writing"}

    # 文章写完了，但还没人工审核
    if s.get("article") and not s.get("human_decision"):
        return {"next_agent": "human_review"}

    # 用户要求重写
    if s.get("human_decision") == "rewrite":
        # 把 human_decision 清空，让写作子图重新跑
        # 同时可以把 previous_feedback 带上
        return {"next_agent": "writing", "human_decision": "", "article": ""}

    # 用户要求终止
    if s.get("human_decision") == "stop":
        return {"next_agent": "FINISH"}

    # 正常继续
    if not s.get("english_version"):
        return {"next_agent": "trans"}
    if not s.get("image_url"):
        return {"next_agent": "image"}
    if not s.get("wechat_url"):
        return {"next_agent": "publish"}

    return {"next_agent": "FINISH"}

下一步方向推荐（三选一）
基于你现在的代码基础，推荐三个递进方向：
A. 多审核点 — 配图后也加人工审核
现在只有写作后暂停，可以在配图后也加一个 human_review，让用户：
确认图片提示词是否满意
不满意可以重新生成提示词/换图片
改动量：小，再写一个 human_review_image 节点 + 主调度加分支即可。
B. 翻译并行化 — 英文日文同时翻译
现在翻译子图里 t_e → t_s → t_j 是串行的。LangGraph 支持 Send 并行派发，可以让英文和日文同时调用模型，缩短等待时间。
改动量：中等，需要把翻译子图改成并行节点，或直接在主图里用 Send 同时触发两个翻译任务。
C. 历史记录与状态回溯 — Time Travel
LangGraph 一个很酷的特性：可以回到任意 checkpoint 重新执行。比如：
用户说"我想回到配图之前，换一张图"
或者"回到翻译前，让 AI 用更正式的语气翻译"
配合 Gradio 界面，可以加一个历史步骤列表，让用户点击"回到某一步"重新执行。
改动量：较大，但功能最强大。
我的建议
如果你想稳扎稳打，选 A（多审核点），半小时就能跑通。
如果你想深入 LangGraph 核心机制，选 B（并行化），能学到 Send 和并行子图。
如果你想做最完整的项目，选 C（状态回溯），这是 LangGraph 区别于其他框架的王牌特性。
