🎯 关键改动
表格
旧代码 新代码

# 第 2 步 — Send 并行执行 multi_agent_Send.py

现在的流程是串行的：Researcher 写完 → Writer 才能写 → Editor 才能审。
并行化场景：如果任务更复杂，比如：
同时让 Researcher 查资料 + Designer 设计配图
或者 Researcher 查多个不同角度的资料（并行调研）
核心概念：Send 对象
Python
from langgraph.types import Send

# 条件边返回 Send 列表，实现并行

def supervisor_node(state): # 同时派发给多个节点，并行执行
return [
Send("researcher", {"topic": state["topic"], "angle": "技术角度"}),
Send("researcher", {"topic": state["topic"], "angle": "教育角度"}),
Send("researcher", {"topic": state["topic"], "angle": "伦理角度"})
]

目标
把串行的 Researcher → Writer → Editor 改成部分并行：
plain
┌→ Researcher_A (技术角度) ┐
Supervisor ──Send──┼→ Researcher_B (教育角度) ┼→ [汇总] → Writer → Editor
└→ Researcher_C (伦理角度) ┘
3 个 Researcher 同时从不同角度调研，然后汇总成一份完整的调研笔记，再交给 Writer。
核心知识点：Send 对象
Python
from langgraph.types import Send

# Send(node_name, state_update)

# 会创建一个新的子图执行，传入 state_update 作为初始状态

Send("researcher", {"topic": "AI教育", "angle": "技术角度"})
关键：条件边返回 Send 列表时，LangGraph 会并行执行所有 Send。
需要修改的地方
表格
改动点 说明
状态加 angle 字段 每个 Researcher 知道自己从什么角度调研
Supervisor 返回 Send 列表 同时派发多个 Researcher
新增 aggregator 节点 把多个调研结果汇总成一份
Researcher 节点读取 angle 根据角度调整 prompt

找到问题了！Send 并行执行后，researcher 节点没有 outgoing edge，所以执行完就停了，aggregator 根本没被触发。
正确的流程应该是：
plain
Supervisor → [Send x3 到 Researcher] → Researcher 执行完回到 Supervisor
↓
Supervisor 检查 partial_notes 数量 ≥ 3 → 路由到 Aggregator
↓
Aggregator 汇总 → 回到 Supervisor → 路由到 Writer
🎯 关键修复
表格
问题 修复
Researcher 执行完没地方去 workflow.add_edge("researcher", "supervisor")
Supervisor 不知道何时汇总 检查 partial_notes 数量 ≥ 3
Aggregator 没被触发 Supervisor 看到 3 个角度后路由到 aggregator
重复汇总 Aggregator 清空 partial_notes

表格
概念 代码体现

子图编译 subgraph.compile()

🔧 修复
修复 1：条件边映射加 "main_supervisor"
Python
workflow.add_conditional_edges(
"main_supervisor",
lambda state: state["next_agent"],
{
"writing_team": "writing_team",
"FINISH": END,
"main_supervisor": "main_supervisor" # ✅ 加上！
}
)
修复 2：打回时也要重置子图内部状态
子图有自己的状态（WritingState），主图 update_state 只改主图状态。需要确保子图重新执行时从干净状态开始。
实际上，由于子图是作为主图的一个节点调用的，每次调用子图时都会传入当前主图状态。只要主图状态中的 draft 和 edit_feedback 被清空，子图重新执行时会看到 research_notes 为空，自然会重新走调研流程。
但 previous_feedback 需要传递进子图，这个已经在主图状态中了，子图节点会读取 state.get("previous_feedback", "")。

# 第 4 步：Stream 实时输出 multi_agent_Stream.py

为什么要改？
表格
invoke() stream()
阻塞等待全部执行完 每执行完一个节点就实时输出
用户看不到进度 用户能看到"正在调研...正在写作..."
适合脚本/后台任务 适合 Web/聊天/交互式应用
Stream 返回的数据格式
Python
for chunk in app.stream(inputs, config=config): # chunk 格式：{节点名: {状态更新}} # 例如：# {"sub_researcher": {"partial_notes": ["【技术角度】..."]}} # {"sub_supervisor": {"next_agent": "aggregator"}}
print(chunk)

📌 Stream 返回的数据结构
Python
for chunk in app.stream(inputs, config=config): # chunk 是一个字典 # { # "sub_researcher": {"partial_notes": ["【技术角度】..."]}, # "sub_supervisor": {"next_agent": "aggregator"} # }

    for node_name, node_output in chunk.items():
        print(f"节点 {node_name} 更新了: {list(node_output.keys())}")
