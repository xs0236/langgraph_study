# 文件 content_factory_timetravel.py

Time Travel 核心原理
LangGraph 的 SqliteSaver 会自动保存每个节点执行后的完整状态快照。你可以：
表格
API 作用
app.get_state_history(config) 获取一个线程的所有历史快照（倒序）
app.update_state(config, values) 在某个历史快照上修改状态值
app.stream(None, config) 从某个历史快照的断点继续执行
相比上一版，改了什么？

1. 新增全局变量 active_config
   Python
   current_config = None # 原始线程（用于刷新历史列表）
   active_config = None # 当前活跃配置（用于 resume / time travel）
2. 抽取通用函数
   Python
   \_process_stream() # 统一处理 stream 事件（中断检测 + 日志）
   \_build_final_yield() # 统一构建最终输出
3. 新增 Time Travel 函数
   Python
   refresh_history() # 读取历史快照，格式化为列表
   time_travel_and_run() # 回到某步骤 → 可选修改状态 → 重新执行
   preset_modify() # 快捷预设：自动生成清空某字段的 JSON
4. 界面新增「⏪ 时间旅行」Tab
   plain
   🔄 刷新历史 ← 点击后加载所有执行步骤
   ─────────────────────────
   [0] m_s | 文章+英文 | cp:abc123...
   [1] writing | 文章 | cp:def456...
   [2] trans | 文章+英文+配图 | cp:ghi789...
   ─────────────────────────
   [Dropdown] 选择回溯点
   [Textbox] 状态修改 JSON（可选）
   [快捷按钮] 📝回到写作前 / 🌐回到翻译前 / 🎨回到配图前 / 📱回到发布前
   [Button] ⏪ 回到此步骤并执行
   使用示例
   场景 1：翻译不满意，想换语气重新翻译
   先正常跑完一次完整流程
   进入「⏪ 时间旅行」Tab，点击 🔄 刷新历史
   选择一个翻译前的步骤（如 trans 节点之前）
   点击快捷按钮 🌐 回到翻译前 → 自动填充：
   JSON
   {"english_version": "", "japanese_version": ""}
   你还可以追加修改，比如：
   JSON
   {"english_version": "", "japanese_version": "", "article": "更正式的语气版本..."}
   点击 ⏪ 回到此步骤并执行
   LangGraph 会从该 checkpoint 继续，因为 english_version 被清空，调度会重新进入 trans 节点重新翻译。
   场景 2：配图不满意，换一张图
   刷新历史，选择配图前的步骤
   点击 🎨 回到配图前（自动清空 image_url 和 image_prompt）
   点击执行，AI 会重新生成提示词和随机图片
   场景 3：发布文案想改风格
   选择发布前的步骤
   点击 📱 回到发布前（清空所有发布链接和文案）
   或者手动在 JSON 里写：
   JSON
   {"weibo_text": "用更活泼的语气写微博..."}
   执行后发布节点会基于新状态重新生成

历史版本多是因为 SqliteSaver 在同一个 team_gradio.db 文件里累积保存了所有运行记录，包括：
之前 content_factory_write.py 跑的记录 — 同一个数据库文件没删
每个节点执行后自动存 checkpoint — 主图的 m_s、writing、trans 等，加上子图内部的 w_r、w_w、w_e 每个都会生成一条
rewrite 会产生分支历史 — 每次回到写作重新跑，都会新增一串 checkpoint
